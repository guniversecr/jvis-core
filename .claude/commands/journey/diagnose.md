# Journey Diagnose

Analyze existing project to detect journey state and completed checkpoints.

## Instructions

Execute the journey diagnosis task to:
1. Scan project for journey artifacts
2. Detect which checkpoints are complete
3. Identify current phase
4. Find blocked items
5. Generate journey state file
6. Provide actionable recommendations

Load and execute: `.jvis/tasks/journey-diagnose.md`

## When to Use

- First time setting up journey tracking
- After significant work without journey updates
- When unsure of project status
- For brownfield projects starting JVIS

## Output

Creates/updates: `docs/journey/journey-state.yaml`
Displays: Full diagnosis report with recommendations
